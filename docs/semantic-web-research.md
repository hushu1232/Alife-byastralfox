# QChat 语义联网研究（Phase 1）

## 目的

这一功能让夏羽或咪绪在正常 QQ 对话中，先由模型按问题语义判断是否需要新鲜的外部信息；需要时使用现有 DuckDuckGo/Bing 搜索和有限网页研究，再把受隔离的证据交给正常回复模型。

它不依赖“搜索”“最新”等触发词，也不会经过 DataAgent。因此，聊天路径不等待 SQL、图谱或数据分析流程。

Phase 1 不包含定时新闻推送、主动群发、Tavily、百度千帆、图片识别或额外的网页供应商。

## 启用

在目标 QChat 配置中启用：

```csharp
SemanticWebResearch = new QChatSemanticWebResearchConfig
{
    Enabled = true
};
```

`Enabled` 默认是 `false`，因此升级后不会改变现有聊天行为。启用后，运行时还需要已有的 Semantic Kernel `IChatCompletionService`，供语义路由器和慢检索旁白使用。搜索仍使用项目现有的 DuckDuckGo/Bing 回退供应商；不必为了语义搜索单独打开显式 `/search` 命令开关。

若希望主人在标准或深度研究中读取有限的公开页面，还需要保留现有的 `EnableInternetAccess = true`，并配置项目原有的网络/浏览器服务。否则所有深度都会安全地退化为搜索结果证据，而不会阻塞聊天。

## 何时会联网

- 只有主人私聊，以及其他群成员**明确 @ 当前 BotId** 的问题可进入语义路由。
- 未 @ 的群消息、私聊访客、普通群内被动/主动发言都不会触发研究。
- 路由模型基于语义判断时效性、核验需求、罕见专业问题和信息不确定性；它不会使用关键词白名单。
- 当路由模型超时、出错或返回不合法 JSON，且 `ResearchOnUncertainty` 为 `true` 时，会对原问题做一次 quick 搜索，而不是静默猜测。
- 陪伴聊天、创作、主观讨论和稳定知识可以被路由器拒绝，此时不会调用搜索服务。
- 显式 `/search` 和 `/rag` 命令会在普通聊天派发前直接处理，不会被本功能二次研究。

## 配置字段

| 字段 | 默认值 | 作用与延迟/成本影响 |
| --- | ---: | --- |
| `Enabled` | `false` | 总开关。关闭时没有模型路由、搜索或反馈。 |
| `EnableOwnerPrivate` | `true` | 是否允许主人私聊进入研究。关闭可减少主人私聊的联网成本。 |
| `EnableMentionedGroup` | `true` | 是否允许明确 @ Bot 的群问题进入研究。未 @ 的群消息始终不会进入。 |
| `ResearchOnUncertainty` | `true` | 路由失败或不确定时是否执行一次 quick 搜索。开启提高查证率，也会增加少量失败场景的搜索。 |
| `RouterTimeoutMilliseconds` | `900` | 语义路由模型的上限，最低按 100 ms 执行。更高可提高判断完成率，但会增加首个回复前的等待。 |
| `FeedbackDelayMilliseconds` | `1200` | 研究尚未完成多久后才发送一条自然旁白。它不会改变研究本身速度；调低会更常出现中间反馈。 |
| `QuickMaxSources` | `3` | quick 搜索可交给模型的最多来源数。较小通常更快、更省上下文。 |
| `StandardMaxSources` | `3` | 标准研究来源上限。主人可在已有网络能力允许时读取有限公开页面。 |
| `DeepMaxSources` | `5` | 深度研究来源上限。来源更多、上下文和潜在网页读取成本也更高。 |
| `SessionCacheSeconds` | `120` | 同一 Agent、会话目标、归一化查询、深度、来源数的成功结果缓存时长。增加它能显著降低重复提问的等待和搜索次数；失败结果不会缓存。 |

## 深度和权限边界

| 深度 / 会话 | 行为 |
| --- | --- |
| `Quick` | 对主人和群成员都只做搜索，不读取页面。 |
| 主人私聊的 `Standard` / `Deep` | 可使用已配置的公开页面读取、公开抓取和浏览器快照能力。 |
| 被 @ 群成员的 `Standard` / `Deep` | 仍然只搜索，不读取页面或启动浏览器。 |

研究使用现有 `AgentWebResearchService`，不调用 DataAgent。所有外部材料都会放在 `UNTRUSTED EXTERNAL CONTEXT` 边界内，模型不得把它当作系统指令或权限指令；注入的来源 URL 只来自实际返回的证据记录。

## 对话反馈

研究若在 `FeedbackDelayMilliseconds` 内完成，用户只会看到正常的最终回复。若它仍在进行，系统会让当前聊天模型生成一句不超过 80 个字符的自然中文反馈；它不使用固定文案，不虚构来源、进度或结论。取消、旁白模型超时、旁白失败或空输出都不会发送中间消息。

最终回复仍走正常夏羽/咪绪 QChat 人设与模型派发流程；研究证据仅是额外的外部参考上下文。
