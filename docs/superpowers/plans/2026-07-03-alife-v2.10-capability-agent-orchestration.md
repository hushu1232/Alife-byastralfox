# Alife V2.10 Capability Agent Orchestration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a lightweight Alife-wide capability governance layer that classifies existing plugins as interaction surfaces, deterministic services, context providers, workflow candidates, or presentation adapters, while adding DataAgent scenario-pack and node-level tool-scope contracts without introducing LangGraph runtime behavior yet.

**Architecture:** V2.10 keeps the current C# plugin architecture as the system of record. It documents and tests which capabilities should be agentized, which must remain deterministic safety services, and how DataAgent workflow nodes will expose narrowly scoped capabilities to reduce attention dilution and random tool choice. LangGraph remains a later optional runtime; V2.10 produces contracts, catalogs, scenario knowledge, readiness gates, and no new execution path.

**Tech Stack:** C#/.NET 9, existing `Alife.Function.FunctionCaller` Tool Broker models, existing `Alife.Function.DataAgent` orchestration/readiness patterns, JSON scenario packs with `System.Text.Json`, NUnit, existing PowerShell readiness scripts.

---

## Scope Lock

V2.10 must do:

- Define an Alife capability governance catalog that classifies existing plugin areas without changing runtime behavior.
- Make the non-overengineering boundary explicit: agent workflows are for long-running, semantic, recoverable chains; deterministic services keep safety and execution authority.
- Add a DataAgent scenario knowledge pack contract for business-term-to-dataset/field/metric mapping.
- Add a DataAgent node-level tool-scope policy contract for planned graph nodes.
- Add readiness and engineering-map gates proving the contracts exist and forbid over-agentizing safety services.
- Keep QChat, DataAgent, Tool Broker, progress, trace, audit, and store behavior unchanged.

V2.10 must not do:

- No LangGraph sidecar process.
- No Python runtime dependency.
- No new SQL execution path.
- No natural-language command auto-execution.
- No frontend ChatBI console.
- No PostgreSQL checkpoint persistence.
- No conversion of QChat, Memory, Vision, Speech, DeskPet, Emotion, SQL safety, audit, or redaction into free-form agents.

```csharp
using System.Text.Json;

namespace Alife.Function.DataAgent;

public static class DataAgentScenarioKnowledgePackProvider
{
    static readonly JsonSerializerOptions Options = new()
    {
        PropertyNameCaseInsensitive = true
    };

    public static DataAgentScenarioKnowledgePack Load(string path)
    {
        ArgumentException.ThrowIfNullOrWhiteSpace(path);

        string json = File.ReadAllText(path);
        DataAgentScenarioKnowledgePack? pack = JsonSerializer.Deserialize<DataAgentScenarioKnowledgePack>(json, Options);
        if (pack is null)
            throw new InvalidOperationException("Scenario knowledge pack could not be parsed.");

        Validate(pack);
        return Snapshot(pack);
    }

    public static IReadOnlyList<DataAgentScenarioTerm> ResolveTerms(
        DataAgentScenarioKnowledgePack pack,
        string utterance)
    {
        ArgumentNullException.ThrowIfNull(pack);
        if (string.IsNullOrWhiteSpace(utterance))
            return [];

        string normalized = utterance.Trim();
        return pack.Terms
            .Where(term => Matches(term, normalized))
            .Select(SnapshotTerm)
            .ToArray();
    }

    static bool Matches(DataAgentScenarioTerm term, string utterance)
    {
        if (utterance.Contains(term.Term, StringComparison.OrdinalIgnoreCase))
            return true;

        return term.Aliases.Any(alias =>
            string.IsNullOrWhiteSpace(alias) == false &&
            utterance.Contains(alias, StringComparison.OrdinalIgnoreCase));
    }

    static void Validate(DataAgentScenarioKnowledgePack pack)
    {
        if (string.IsNullOrWhiteSpace(pack.Scenario))
            throw new InvalidOperationException("Scenario knowledge pack scenario cannot be blank.");

        if (string.IsNullOrWhiteSpace(pack.Culture))
            throw new InvalidOperationException("Scenario knowledge pack culture cannot be blank.");

        HashSet<string> terms = new(StringComparer.Ordinal);
        foreach (DataAgentScenarioTerm term in pack.Terms ?? [])
        {
            if (string.IsNullOrWhiteSpace(term.Term))
                throw new InvalidOperationException("Scenario term cannot be blank.");

            if (!terms.Add(term.Term))
                throw new InvalidOperationException($"Duplicate scenario term '{term.Term}'.");

            if (string.IsNullOrWhiteSpace(term.Dataset))
```

```csharp
    public const string BuildEvidencePack = "dataagent.evidence.build";
    public const string WriteAudit = "dataagent.audit.write";
    public const string PublishProgress = "dataagent.progress.publish";
    public const string ReadProgressDiagnostics = "dataagent.diagnostics.progress.read";
    public const string ReadTraceDiagnostics = "dataagent.diagnostics.trace.read";
    public const string ReadEvidenceDiagnostics = "dataagent.diagnostics.evidence.read";
}

public sealed record DataAgentNodeToolScope(
    string NodeName,
    bool AllowsModelCall,
    IReadOnlyList<string> AllowedCapabilities,
    string Reason);

public static class DataAgentToolScopePolicy
{
    static readonly IReadOnlyList<DataAgentNodeToolScope> DefaultScopes =
        Array.AsReadOnly(
        [
            Scope(DataAgentWorkflowNodeNames.RouteGate, false, [], "route_gate_uses_tool_broker_state"),
            Scope(DataAgentWorkflowNodeNames.ScenarioKnowledge, false,
                [DataAgentNodeCapabilities.ReadCatalog, DataAgentNodeCapabilities.ReadScenarioPack],
                "scenario_knowledge_is_read_only_context"),
            Scope(DataAgentWorkflowNodeNames.QueryPlanner, true,
                [
                    DataAgentNodeCapabilities.ReadCatalog,
                    DataAgentNodeCapabilities.ReadScenarioPack,
                    DataAgentNodeCapabilities.GenerateQueryPlan
                ],
                "planner_generates_query_plan_only"),
            Scope(DataAgentWorkflowNodeNames.QueryPlanValidator, false,
                [DataAgentNodeCapabilities.ValidateQueryPlan],
                "validator_is_deterministic"),
            Scope(DataAgentWorkflowNodeNames.SqlCompiler, false,
                [DataAgentNodeCapabilities.CompileSql],
                "compiler_is_deterministic"),
            Scope(DataAgentWorkflowNodeNames.SqlSafety, false,
                [DataAgentNodeCapabilities.ValidateSqlSafety],
                "sql_safety_is_deterministic"),
            Scope(DataAgentWorkflowNodeNames.ReadOnlyExecute, false,
                [DataAgentNodeCapabilities.ExecuteReadOnlyQuery],
                "execution_is_read_only_tool_node"),
            Scope(DataAgentWorkflowNodeNames.ResultExplainer, true,
                [DataAgentNodeCapabilities.ExplainResult],
                "explainer_cannot_plan_or_execute_queries"),
            Scope(DataAgentWorkflowNodeNames.EvidenceAudit, false,
                [DataAgentNodeCapabilities.BuildEvidencePack, DataAgentNodeCapabilities.WriteAudit],
                "evidence_and_audit_are_deterministic"),
            Scope(DataAgentWorkflowNodeNames.CheckpointProgress, false,
                [DataAgentNodeCapabilities.PublishProgress],
                "checkpoint_progress_is_deterministic"),
            Scope(DataAgentWorkflowNodeNames.DiagnosticsRouter, true,
                [
                    DataAgentNodeCapabilities.ReadProgressDiagnostics,
                    DataAgentNodeCapabilities.ReadTraceDiagnostics,
                    DataAgentNodeCapabilities.ReadEvidenceDiagnostics
                ],
                "diagnostics_are_owner_only_read_only")
        ]);

    public static IReadOnlyList<DataAgentNodeToolScope> CreateDefault()
    {
        return DefaultScopes.Select(Snapshot).ToArray();
    }

    public static DataAgentNodeToolScope ForNode(string nodeName)
    {
        if (string.IsNullOrWhiteSpace(nodeName))
            return Scope(string.Empty, false, [], "unknown_node_fail_closed");

        DataAgentNodeToolScope? scope = DefaultScopes.FirstOrDefault(item =>
            item.NodeName.Equals(nodeName.Trim(), StringComparison.Ordinal));
        return scope is null
            ? Scope(nodeName.Trim(), false, [], "unknown_node_fail_closed")
            : Snapshot(scope);
    }

    static DataAgentNodeToolScope Scope(
        string nodeName,
        bool allowsModelCall,
        IReadOnlyList<string> allowedCapabilities,
        string reason)
    {
        return new DataAgentNodeToolScope(
            nodeName,
            allowsModelCall,
            allowedCapabilities.ToArray(),
            reason);
    }

    static DataAgentNodeToolScope Snapshot(DataAgentNodeToolScope scope)
    {
        return scope with
        {
            AllowedCapabilities = scope.AllowedCapabilities.ToArray()
        };
    }
}
```

```markdown
# Alife Capability Agent Orchestration Design

## Purpose

V2.10 does not add LangGraph runtime behavior. It makes Alife's existing plugin system governable by classifying which capabilities are interaction surfaces, deterministic services, context providers, workflow candidates, perception adapters, presentation adapters, or external bridges.

## Non-Overengineering Rule

Do not agentize a capability just because it exists. Agent workflows are reserved for long-running, semantic, recoverable chains such as DataAgent query analysis, owner diagnostics routing, future web research, and future desktop stewardship. Safety checks, SQL validation, read-only execution, audit, redaction, file policies, approval gates, perception adapters, and presentation adapters remain deterministic services.

## Current System Roles

- QChat is the interaction and owner-command surface.
- FunctionCaller is the Tool Broker and execution policy layer.
- DataAgent is the first workflow candidate because it already has route, plan, validate, execute, explain, evidence, checkpoint, progress, and trace boundaries.
- Browser and DesktopControl are future workflow candidates behind owner gates and approval policies.
- Memory is a context provider.
- Vision, Speech, and Auditory are perception adapters.
- DeskPet, Emotion, and VirtualWorld are presentation adapters.
- MCP, Python, and Developer services are external or high-risk bridges.

## V2.10 Deliverables

- Alife capability governance catalog.
- DataAgent engineering scenario knowledge pack.
- DataAgent node-level tool-scope policy.
- Readiness and engineering-map gates.

## Later Work

V2.11 wires DataAgent scenario-pack context into planner prompts and owner diagnostics. V2.12 activates that scenario context in the real DataAgent runtime path. V2.13 productizes optional PostgreSQL checkpoint/session persistence for recoverable DataAgent analysis chains. V2.14 may add the disabled-by-default LangGraph sidecar contract. V2.15 may pilot a DataQueryGraph that calls C# safety services instead of bypassing them.
```
